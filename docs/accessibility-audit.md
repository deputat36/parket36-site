# Автоматический accessibility-аудит

Дата обновления: 2026-07-10.

Аудит запускается внутри workflow `.github/workflows/browser-smoke.yml` вместе с Playwright smoke-тестами.

Файл тестов: `tests/e2e/accessibility.spec.mjs`.

## Проверяемые страницы

- главная `/`;
- форма оценки `/zayavka/`;
- страница ошибки `/404.html`.

## Стандарт проверки

Используется `@axe-core/playwright` и правила:

- WCAG 2.0 A;
- WCAG 2.0 AA;
- WCAG 2.1 A;
- WCAG 2.1 AA.

Тест завершается ошибкой при любом обнаруженном нарушении из этих наборов правил. В сообщении отображаются:

- идентификатор правила axe;
- уровень влияния;
- описание нарушения;
- CSS-селекторы проблемных элементов.

## Изоляция

Аудит выполняется против локально собранной папки `_site`. Он не зависит от DNS, GitHub Pages или production Supabase.

## Локальный запуск

```bash
npm install --no-audit --no-fund
npx playwright install chromium
npm run test:e2e -- tests/e2e/accessibility.spec.mjs
```

## Что не заменяет axe

Автоматический аудит не может полностью проверить:

- понятность текстов и подписей;
- логичность порядка чтения;
- удобство клавиатурной навигации в сложных сценариях;
- качество озвучивания конкретными экранными дикторами;
- визуальное восприятие на реальных устройствах.

После существенного изменения меню, формы, цветов или структуры страницы нужна дополнительная ручная проверка клавиатурой.
