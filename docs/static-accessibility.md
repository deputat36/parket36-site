# Обязательная доступность в готовом HTML

Дата обновления: 2026-07-10.

## Цель

Критические элементы навигации не должны зависеть от выполнения JavaScript. Даже если скрипт заблокирован, не загрузился или отключён пользователем, готовая страница должна содержать корректные landmark и связи элементов меню.

## Что добавляется при сборке

`tools/html_accessibility.py` обрабатывает каждый HTML-файл в `_site` и обеспечивает:

- одну ссылку пропуска к основному содержанию;
- `href="#main-content"` у skip-link;
- `id="main-content"` у элемента `<main>`;
- `aria-controls="site-navigation"` у кнопки мобильного меню;
- начальное `aria-expanded="false"` у кнопки меню;
- `id="site-navigation"` у связанного `<nav>`.

Исходные HTML-файлы не переписываются. Обязательная разметка появляется в публичной сборке до публикации GitHub Pages.

## JavaScript остаётся улучшением

`js/main.js` по-прежнему:

- меняет `aria-expanded` при открытии меню;
- закрывает меню по `Escape`;
- добавляет активное состояние текущего раздела;
- управляет динамическими элементами интерфейса.

Но исходное доступное состояние уже существует в HTML и не создаётся JavaScript-ом с нуля.

## Автоматические проверки

Сборка завершается ошибкой, если:

- отсутствует `<body>` или `<main>`;
- skip-link отсутствует или продублирован;
- `main-content` отсутствует или продублирован;
- кнопка меню и навигация присутствуют не вместе;
- ARIA-связь меню имеет другое значение.

Playwright-тест `tests/e2e/no-js-accessibility.spec.mjs` запускает главную и форму с `javaScriptEnabled: false` и проверяет:

- статический skip-link;
- landmark `main#main-content`;
- ARIA-связь меню;
- доступность телефонной ссылки;
- наличие CSS bundle;
- сохранение формы в HTML.

## Локальная проверка

```bash
python tools/build_pages.py
npm run test:e2e -- tests/e2e/no-js-accessibility.spec.mjs
```

Автоматический no-JS тест не заменяет ручную проверку клавиатурой и экранным диктором, но блокирует возврат зависимости базовой навигации от JavaScript.
