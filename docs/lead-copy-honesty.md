# Честный текст заявки

## Принцип

Публичный текст не смешивает три разных технических факта:

1. форма пытается сохранить заявку в защищённой системе;
2. backend отдельно сообщает, подтверждено ли автоматическое уведомление Ивану;
3. готовый текст копируется или показывается для ручной передачи.

Формулировки «заявка уйдёт Ивану», «сайт отправит заявку Ивану» и аналогичные безусловные обещания запрещены, если рядом нет фактического статуса `notification: sent`.

## Где применяется

Нормализация выполняется модулем `tools/lead_copy.py` для HTML и JavaScript внутри публичной сборки `_site` до fingerprinting.

Это важно для `js/main.js`: старый динамический текст не должен повторно появляться после загрузки страницы, даже если HTML уже исправлен.

Проверяются страницы:

- `/zayavka/`;
- `/kontakty/`;
- `/politika/`;
- главная и другие страницы, содержащие форму или общий `main.js`.

## Текст для посетителя

Посетитель заранее видит понятную последовательность:

- сервис попробует сохранить заявку;
- если уведомление Ивану не подтвердится, форма сразу покажет номер и кнопку звонка;
- текст заявки останется доступным для ручной передачи вместе с фотографиями.

## Защита от регрессии

- `tools/lead_copy.py` содержит централизованные замены и запрещённые маркеры;
- `tools/check_lead_copy_honesty.py` проверяет интеграцию до fingerprinting, документацию и E2E;
- `tests/e2e/lead-copy-honesty.spec.mjs` проверяет итоговый текст в Chromium;
- проверка не читает контактные данные, не вызывает production endpoint и не меняет аналитику.
