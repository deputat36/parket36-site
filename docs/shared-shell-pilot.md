# Общая оболочка страниц: профили

Дата обновления: 2026-07-15.

## Принцип

`tools/shared_shell.py` применяет общие HTML-fragments только при сборке `_site`. Исходные HTML остаются полноценным читаемым fallback.

Для каждой страницы профиль явно определяет:

- подключаемые компоненты;
- активный пункт основной навигации или его намеренное отсутствие;
- адрес и подпись второго действия mobile CTA.

Локальный контент, SEO-метаданные, structured data, canonical, тематические футеры и уникальные CTA не заменяются, если они не перечислены в `components`.

## Явные профили

Полный профиль оболочки:

- `/`;
- `/zayavka/`.

Он включает header, final CTA, footer и mobile CTA.

Профиль основной навигации:

- `/uslugi/`;
- `/ceny/`;
- `/o-mastere/`;
- `/portfolio/`;
- `/sovety/`;
- `/kontakty/`.

Он включает header, footer и mobile CTA. Уникальный final CTA сохраняется локальным. Активная ссылка получает одновременно `active` и `aria-current="page"`.

Вспомогательные верхнеуровневые страницы:

- `/resheniya/`;
- `/voprosy-i-otvety/`;
- `/kak-rabotaem/`.

Они используют header, footer и mobile CTA без активного пункта основной навигации.

## Основные услуги

Точный профиль применяется к восьми основным страницам:

- `/uslugi/parket-i-poly/`;
- `/uslugi/ciklevka-parketa/`;
- `/uslugi/restavraciya-parketa/`;
- `/uslugi/ukladka-parketa/`;
- `/uslugi/ukladka-laminata/`;
- `/uslugi/shlifovka-doshchatogo-pola/`;
- `/uslugi/terrasy-i-derevyannye-poly/`;
- `/uslugi/pokrytie-lakom-i-maslom/`.

Общими становятся только header и mobile CTA. Активируется родительский пункт `/uslugi/`. Тематические footer и final CTA сохраняются локальными.

## Семейство статей

Автоматически обнаруживаются все прямые страницы `sovety/*/index.html`.

Контракт семейства:

- ровно один `og:type=article`;
- общие header и mobile CTA;
- активный родительский пункт `/sovety/`;
- локальные footer, материал и возможный final CTA.

Новая прямая статья автоматически получает профиль. Неожиданный вложенный уровень блокирует сборку до отдельного правила.

## Семейство готовых решений

Автоматически обнаруживаются все прямые страницы `resheniya/*/index.html`.

Контракт семейства:

- ровно один `og:type=website`;
- общие header и mobile CTA;
- ни один пункт основной навигации не активируется;
- локальные footer, основной материал и final CTA сохраняются.

## Смежные noindex-услуги

После отдельного аудита к shared shell подключены девять страниц:

- `/uslugi/demontazh/`;
- `/uslugi/elektrika/`;
- `/uslugi/melkiy-remont/`;
- `/uslugi/muzh-na-chas/`;
- `/uslugi/otdelka/`;
- `/uslugi/pereezdy/`;
- `/uslugi/santehnika/`;
- `/uslugi/sborka-mebeli/`;
- `/uslugi/vyvoz-musora/`.

Это семейство задаётся точным allowlist, а не glob-маской. Новая смежная страница не подключается автоматически.

Обязательный контракт каждой страницы:

- путь находится непосредственно под `/uslugi/`;
- ровно один `og:type=website`;
- ровно один `robots=noindex, follow`;
- общими становятся только header и mobile CTA;
- активный пункт основной навигации отсутствует;
- тематические footer и final CTA сохраняются локальными.

Пункт `/uslugi/` намеренно не подсвечивается. Эти страницы объясняют границы смежных вопросов и не должны выглядеть как основные коммерческие услуги Паркет36.

`/uslugi/master-na-chas/` не входит в семейство. Это отдельный noindex redirect-переходник на `/uslugi/muzh-na-chas/` с упрощённой шапкой, `meta refresh` и без final CTA.

## Общие fragments

Едиными источниками служат:

- `data/shared-shell/header.htmlfrag`;
- `data/shared-shell/final-cta.htmlfrag`;
- `data/shared-shell/footer.htmlfrag`;
- `data/shared-shell/mobile-cta.htmlfrag`.

Расширение `.htmlfrag` не позволяет статическому SEO-аудиту принять fragment за самостоятельную публичную страницу.

## Fail-closed проверки семейств

Каждое семейство задаёт:

- уникальное имя;
- безопасный относительный корень;
- либо безопасную прямую маску `*/index.html`, либо точный tuple `members`;
- обязательный `og:type`;
- при необходимости обязательное значение `robots`;
- профиль компонентов и навигации.

Сборка блокируется при:

- пустом или небезопасном корне;
- одновременном использовании `glob` и `members`;
- произвольной glob-маске;
- пустом allowlist;
- дублирующихся или вложенных allowlist-путях;
- отсутствии страницы в source или `_site`;
- расхождении автоматически обнаруженного семейства между source и `_site`;
- неправильном `og:type` или обязательном `robots`;
- пересечении явных и семейных профилей;
- повторяющихся именах или корнях семейств.

## Гарантии компонентов

Для каждой профильной страницы сборка проверяет:

- существование fragment и уникального marker;
- известные компоненты без дублей;
- ровно один заменяемый блок каждого подключаемого типа;
- ровно один rendered fragment в результате;
- однозначный active nav или его намеренное отсутствие;
- существующую цель и непустую подпись mobile CTA;
- отсутствие замены компонентов, которых нет в профиле.

## Браузерные проверки

Playwright запускается с отключённым JavaScript и viewport `390×844`.

Проверяются:

- совпадение полной оболочки `/` и `/zayavka/`;
- основная навигация и `aria-current`;
- отсутствие ложной подсветки вспомогательных страниц и решений;
- все восемь основных услуг;
- представители семейства статей;
- все пять решений;
- все девять смежных noindex-услуг;
- сохранение их локальных footer и final CTA;
- `noindex, follow` и `og:type=website` смежных страниц;
- отдельный redirect-контракт `/uslugi/master-na-chas/`;
- телефон и действие `/zayavka/`;
- shared markers только у подключённых компонентов.
