# Общая оболочка страниц: пилот

Дата обновления: 2026-07-10.

Пилотные страницы:

- `/`;
- `/zayavka/`.

## Общие fragments

Единым источником для обеих страниц служат:

- `data/shared-shell/header.htmlfrag`;
- `data/shared-shell/final-cta.htmlfrag`;
- `data/shared-shell/footer.htmlfrag`;
- `data/shared-shell/mobile-cta.htmlfrag`.

Расширение `.htmlfrag` используется специально: статический SEO-аудит не должен воспринимать fragment как самостоятельную публичную HTML-страницу.

Во время сборки `tools/shared_shell.py` заменяет соответствующие блоки только в сгенерированной папке `_site`. Исходные HTML-страницы пока сохраняются как полный читаемый fallback, поэтому переход можно продолжать постепенно и без массовой миграции.

## Что гарантирует сборка

- каждый fragment существует и содержит уникальный marker;
- на каждой пилотной странице находится ровно один заменяемый блок каждого типа;
- итоговая страница содержит ровно один canonical fragment;
- обязательная ARIA-разметка добавляется после применения общей оболочки;
- fragments из `data/` не попадают в публичную сборку как отдельные файлы.

## Браузерная проверка

`tests/e2e/shared-shell.spec.mjs` открывает главную и `/zayavka/` с отключённым JavaScript и сравнивает итоговый HTML:

- шапки;
- финального CTA;
- футера;
- мобильной панели действий.

Тест блокирует PR, если оболочки снова начинают расходиться.

## Следующее расширение

После стабильного пилота можно подключать следующие коммерческие страницы небольшими группами. Сначала нужно выбирать страницы с одинаковой логикой CTA и проверять их через Playwright, а не переводить весь сайт одним большим изменением.
