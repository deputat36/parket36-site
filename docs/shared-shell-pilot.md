# Общая оболочка страниц: профили

Дата обновления: 2026-07-15.

## Подключённые страницы

Полный профиль оболочки:

- `/`;
- `/zayavka/`.

Он включает:

- шапку;
- финальный CTA;
- футер;
- мобильную панель действий.

Профиль страницы основной навигации:

- `/uslugi/`;
- `/ceny/`;
- `/o-masterе/`;
- `/portfolio/`;
- `/sovety/`;
- `/kontakty/`.

Он включает:

- шапку;
- футер;
- мобильную панель действий.

Уникальный финальный CTA этих страниц не заменяется. Для `/o-masterе/`, `/portfolio/`, `/sovety/` и `/kontakty/` его заголовок отдельно закреплён браузерным тестом, чтобы постепенная унификация оболочки не меняла смысл страницы.

Профиль вспомогательной верхнеуровневой страницы:

- `/resheniya/`;
- `/voprosy-i-otvety/`;
- `/kak-rabotaem/`.

Он также включает шапку, футер и мобильную панель действий, но не выделяет ни один пункт основной навигации. Уникальные финальные CTA этих страниц не заменяются и закреплены браузерным тестом.

Профиль страницы услуги:

- `/uslugi/parket-i-poly/`;
- `/uslugi/ciklevka-parketa/`;
- `/uslugi/restavraciya-parketa/`;
- `/uslugi/ukladka-parketa/`;
- `/uslugi/ukladka-laminata/`;
- `/uslugi/shlifovka-doshchatogo-pola/`;
- `/uslugi/terrasy-i-derevyannye-poly/`;
- `/uslugi/pokrytie-lakom-i-maslom/`.

Он включает только:

- шапку;
- мобильную панель действий.

На всех страницах услуг активируется родительский пункт `/uslugi/`. Тематический футер и уникальный финальный CTA намеренно не заменяются: они содержат контекстные ссылки и формулировки конкретной услуги. Браузерный тест проверяет отсутствие shared markers у этих двух блоков и закрепляет их собственные тексты.

Семейный профиль статьи совета:

- все прямые страницы вида `/sovety/{slug}/`;
- индекс `/sovety/` остаётся отдельным явным профилем основной навигации.

Семейный профиль страницы решения:

- все прямые страницы вида `/resheniya/{slug}/`;
- индекс `/resheniya/` остаётся отдельным явным профилем вспомогательной верхнеуровневой страницы.

Оба семейных профиля включают только:

- шапку;
- мобильную панель действий.

Статьи и решения обнаруживаются автоматически по исходным файлам `sovety/*/index.html` и `resheniya/*/index.html`. На статьях активируется родительский пункт `/sovety/`. На решениях ни один пункт основной навигации не выделяется, потому что `/resheniya/` не входит в основное меню. Локальные футеры, основной материал и уникальные CTA не заменяются. Добавление новой прямой страницы семейства не требует ручного изменения `PAGE_PROFILES`.

## Общие fragments

Едиными источниками служат:

- `data/shared-shell/header.htmlfrag`;
- `data/shared-shell/final-cta.htmlfrag`;
- `data/shared-shell/footer.htmlfrag`;
- `data/shared-shell/mobile-cta.htmlfrag`.

Расширение `.htmlfrag` используется специально: статический SEO-аудит не воспринимает fragment как самостоятельную публичную HTML-страницу.

Во время сборки `tools/shared_shell.py` объединяет явные профили из `PAGE_PROFILES` с автоматически обнаруженными семействами из `FAMILY_PROFILES` и заменяет только перечисленные компоненты внутри сгенерированной папки `_site`. Исходные HTML-страницы сохраняются как полный читаемый fallback, поэтому переход продолжается без массовой миграции.

## Профиль страницы

Каждая подключённая страница определяет:

- `components` — какие части общей оболочки применять;
- `active_nav` — какой пункт меню активен или `None`, если страница не является пунктом основной навигации;
- `request_href` — куда ведёт второе действие в мобильной панели;
- `request_label` — необязательная подпись второго действия, по умолчанию «Оценка по фото».

На `/` и `/zayavka/` используется локальная цель `#request`.

На остальных подключённых страницах без встроенной формы используется `/zayavka/`.

На `/kontakty/` сохраняется профильное действие «Обратный звонок» со ссылкой `#callback`. Сборка и браузерный тест отдельно проверяют существование соответствующей секции.

Активная ссылка меню получает одновременно:

- класс `active`;
- `aria-current="page"`.

На вспомогательных страницах и страницах решений `active_nav` равен `None`, поэтому общая шапка остаётся без ложной подсветки раздела.

На всех вложенных страницах услуг `active_nav` равен `/uslugi/`, поэтому пользователь видит родительский раздел без добавления нового пункта меню.

На всех прямых статьях семейства `active_nav` равен `/sovety/`.

## Как обнаруживаются семейства страниц

Каждое семейство декларативно задаёт:

- уникальное имя;
- безопасный относительный корень;
- единственную разрешённую маску `*/index.html`;
- обязательное значение `og:type`;
- применяемый профиль компонентов и навигации.

Сборка отдельно сравнивает исходное дерево и `_site`:

- исходные прямые страницы семейства должны полностью совпасть со страницами в публичной сборке;
- каждая статья `/sovety/{slug}/` должна содержать ровно один `og:type=article`;
- каждая страница `/resheniya/{slug}/` должна содержать ровно один `og:type=website`;
- семейная страница не может одновременно иметь явный или другой семейный профиль;
- имена и корни семейств не могут дублироваться;
- абсолютные пути, `..` и произвольные glob-маски запрещены;
- неожиданные вложенные страницы вида `{root}/{group}/{slug}/index.html` запрещены до добавления отдельного осознанного правила;
- пустое или отсутствующее семейство считается ошибкой сборки.

Так новая прямая страница автоматически попадает под нужный профиль, а структурное исключение не публикуется молча.

## Отчёт покрытия

`tools/build_shared_shell_coverage.py` классифицирует каждую HTML-страницу, которая реально входит в публичную сборку:

- `explicit` — явный профиль из `PAGE_PROFILES`;
- `family:{name}` — автоматически обнаруженный семейный профиль;
- `outside` — страница пока не использует shared shell.

Текущий baseline:

- публичных HTML-страниц — 107;
- страниц с профилем — 94;
- страниц вне shared shell — 13;
- покрытие — 87,9%.

Вне shared shell намеренно остаются:

- `404.html`;
- `/politika/`;
- `/pozvonit-ivanu/`;
- десять дополнительных noindex-страниц смежных услуг.

Эти страницы нельзя автоматически объединять с основными паркетными услугами только ради роста процента покрытия. Сначала нужно отдельно проверить их назначение, индексирование, навигацию, footer и CTA.

Сохранённые файлы:

- `docs/shared-shell-coverage.md` — сводка и полный список страниц вне профилей;
- `docs/shared-shell-coverage.csv` — классификация всех публичных HTML.

`tools/check_shared_shell_coverage.py` повторно генерирует оба файла во временной папке и блокирует quality gate, если изменились публичные страницы, явные профили, семейства или их параметры, а сохранённый baseline не обновлён. Workflow `Site quality` дополнительно публикует свежий artifact `shared-shell-coverage` на 30 дней.

## Что гарантирует сборка

- каждый fragment существует и содержит уникальный marker;
- профиль содержит только известные компоненты без дублей;
- на странице находится ровно один заменяемый блок каждого подключённого типа;
- итоговая страница содержит ровно один rendered fragment;
- активный пункт меню определяется однозначно или намеренно отсутствует;
- мобильная CTA получает существующую цель и непустую подпись;
- профильная подпись не меняет canonical fragment;
- компоненты, отсутствующие в `components`, не заменяются;
- все прямые статьи `/sovety/*/` и решения `/resheniya/*/` охвачены семейными профилями;
- исходные и публичные наборы страниц каждого семейства совпадают;
- обязательная ARIA-разметка добавляется после общей оболочки;
- fragments из `data/` не попадают в публичную сборку как отдельные файлы.

## Браузерная проверка

`tests/e2e/shared-shell.spec.mjs` запускается с отключённым JavaScript.

Проверяется:

- полное совпадение оболочки `/` и `/zayavka/`;
- единый футер на страницах, где он входит в профиль;
- активный пункт меню и `aria-current` на всех шести страницах основной навигации;
- отсутствие `active` и `aria-current` на `/resheniya/`, `/voprosy-i-otvety/` и `/kak-rabotaem/`;
- активный родительский пункт `/uslugi/` на всех восьми страницах услуг;
- активный родительский пункт `/sovety/` на представителях семейства статей;
- отсутствие ложной подсветки меню на всех пяти страницах решений;
- базовая статья, фото-чек-лист с дополнительным CSS и новый материал перед лакировкой;
- собственные финальные CTA и тематические футеры всех пяти решений;
- рабочая телефонная ссылка;
- корректная ссылка `/zayavka/` и подпись «Оценка по фото» на страницах без встроенной формы;
- ссылка `#callback`, подпись «Обратный звонок» и существование callback-секции на `/kontakty/`;
- отсутствие ошибочной ссылки `#request` на профильных страницах;
- сохранение уникальных финальных CTA;
- сохранение тематических футеров на страницах услуг, статьях и решениях;
- наличие shared markers только у подключённых компонентов;
- мобильная панель проверяется на viewport `390×844`.

## Следующее расширение

Основная навигация, вспомогательные верхнеуровневые страницы, весь публичный раздел услуг, семейство статей советов и семейство готовых решений охвачены профильной оболочкой. Отчёт покрытия теперь показывает оставшиеся исключения и должен использоваться как источник следующего решения. Массово подключать noindex-страницы смежных услуг нельзя: каждую группу нужно сначала проверить по назначению, навигации, футеру, CTA и коммерческому фокусу.
