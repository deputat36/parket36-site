# Тема обратного звонка по первой странице

Короткая форма на `/kontakty/` остаётся общей, но для трёх коммерческих входов автоматически уточняет тему обращения:

| Первая страница | Тема |
|---|---|
| `/ceny/` | стоимость паркетных работ |
| `/uslugi/ciklevka-parketa/` | циклёвка и шлифовка паркета |
| `/uslugi/restavraciya-parketa/` | реставрация и ремонт паркета |

## Как работает

`js/main.js` сохраняет первую посадочную страницу в `sessionStorage` как часть first-touch атрибуции. После перехода на `/kontakty/#callback` файл `js/callback-form.js` читает только это сохранённое значение и сопоставляет его с внутренним разрешённым списком.

URL-параметр с произвольной темой не используется. Посетитель не может подставить собственный текст в скрытые поля заявки.

Если первая страница не входит в список, форма сохраняет общий текст консультации по паркетным работам.

## Что видит посетитель

Для распознанной темы под заголовком формы появляется пояснение, например:

```text
Тема обращения: циклёвка и шлифовка паркета.
```

Форма получает атрибут `data-callback-topic`, который помогает браузерным тестам и может использоваться для безопасного оформления.

## Что получает Иван

Поле `service` остаётся единым:

```text
Обратный звонок по паркетным работам
```

Поле `task` становится конкретным. Для циклёвки в заявке сохраняется просьба обсудить циклёвку или шлифовку и подсказать необходимые фото или видео. Это не требует новой колонки и не меняет Edge Function.

## Аналитика

События `callback-open` и `callback-request` получают поле `topic`:

```text
general
stoimost
cyclevka
restavraciya
```

В `dataLayer` значение передаётся как `callback_topic`.

## Защита

`tools/check_callback_topic_context.py` проверяет:

- три разрешённые landing-страницы;
- чтение first-touch записи;
- общий fallback;
- видимое пояснение темы;
- поле `callback_topic` в аналитике;
- наличие трёх Playwright-сценариев;
- отсутствие чтения произвольного `topic` из URL.

## Браузерные сценарии

`tests/e2e/callback-topic-context.spec.mjs` подтверждает:

1. прямой вход на контакты остаётся общей консультацией;
2. переход со стоимости показывает тему бюджета;
3. переход с циклёвки отправляет конкретный текст задачи;
4. UTM и first-touch landing сохраняются;
5. события открытия и отправки содержат правильный `callback_topic`.

## Ограничения

- тема помогает первичной обработке, но не заменяет описание состояния пола;
- точная цена, состав работ и возможность выезда подтверждаются только после разговора;
- при недоступном `sessionStorage` форма безопасно остаётся общей.
