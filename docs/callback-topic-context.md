# Тема обратного звонка по коммерческому контексту

Короткая форма на `/kontakty/` остаётся общей, но для трёх коммерческих входов автоматически уточняет тему обращения:

| Страница контекста | Тема |
|---|---|
| `/ceny/` | стоимость паркетных работ |
| `/uslugi/ciklevka-parketa/` | циклёвка и шлифовка паркета |
| `/uslugi/restavraciya-parketa/` | реставрация и ремонт паркета |

## Как выбирается тема

Приоритет источников:

1. внутренняя страница, с которой посетитель непосредственно перешёл на `/kontakty/#callback`;
2. первая посадочная страница из first-touch атрибуции;
3. общий текст консультации, если ни один путь не входит в разрешённый список.

`document.referrer` используется только при совпадении `origin` с текущим сайтом. Внешний referrer не может изменить тему заявки.

`js/main.js` сохраняет первую посадочную страницу в `sessionStorage` как часть first-touch атрибуции. Этот путь используется как fallback, когда браузер не передал внутренний referrer.

URL-параметр с произвольной темой не используется. Посетитель не может подставить собственный текст в скрытые поля заявки.

## Что видит посетитель

Для распознанной темы под заголовком формы появляется пояснение, например:

```text
Тема обращения: циклёвка и шлифовка паркета.
```

Форма получает атрибуты:

```text
data-callback-topic="cyclevka"
data-callback-topic-source="referrer"
```

Они помогают браузерным тестам и могут использоваться для безопасного оформления.

## Что получает Иван

Поле `service` остаётся единым:

```text
Обратный звонок по паркетным работам
```

Поле `task` становится конкретным. Для циклёвки в заявке сохраняется просьба обсудить циклёвку или шлифовку и подсказать необходимые фото или видео. Это не требует новой колонки и не меняет Edge Function.

## Аналитика

События `callback-open` и `callback-request` получают поля:

```text
topic: general | stoimost | cyclevka | restavraciya
topicSource: general | referrer | first-touch
```

В `dataLayer` значения передаются как:

```text
callback_topic
callback_topic_source
```

## Защита

`tools/check_callback_topic_context.py` проверяет:

- три разрешённых коммерческих пути;
- ограничение referrer текущим origin;
- приоритет текущей коммерческой страницы;
- first-touch fallback;
- общий fallback;
- видимое пояснение темы;
- поля `callback_topic` и `callback_topic_source` в аналитике;
- наличие четырёх Playwright-сценариев;
- отсутствие чтения произвольного `topic` из URL.

## Браузерные сценарии

`tests/e2e/callback-topic-context.spec.mjs` подтверждает:

1. прямой вход на контакты остаётся общей консультацией;
2. first-touch восстанавливает тему без внутреннего referrer;
3. переход со стоимости показывает тему бюджета;
4. переход с циклёвки отправляет конкретный текст задачи;
5. UTM и first-touch landing сохраняются;
6. события открытия и отправки содержат правильные тему и источник темы.

## Ограничения

- тема помогает первичной обработке, но не заменяет описание состояния пола;
- точная цена, состав работ и возможность выезда подтверждаются только после разговора;
- при недоступном `sessionStorage` и отсутствии безопасного referrer форма остаётся общей.
