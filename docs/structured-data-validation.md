# Проверка структурированных данных

## Цель

JSON-LD на публичных страницах должен оставаться валидным, согласованным с canonical и не содержать неподтверждённых рейтингов, отзывов, офферов или цен.

## Что проверяется

`tools/check_structured_data.py` проходит по всем публичным исходным HTML-страницам и проверяет:

- корректность JSON каждого блока `application/ld+json`;
- контекст `https://schema.org`;
- абсолютные URL на домене `https://parket36.ru`;
- совпадение `Article.mainEntityOfPage`, `Service.url` и `WebPage.url` с canonical;
- наличие непустых H1 и `Article.headline` без требования дословного совпадения;
- формат `datePublished` и `dateModified` как `YYYY-MM-DD`;
- отсутствие даты изменения раньше даты публикации;
- наличие автора и издателя статьи;
- наличие поставщика и единого телефона у `Service`;
- корректность `serviceType` и `areaServed`, если эти необязательные поля указаны;
- структуру вопросов и ответов `FAQPage`;
- единый телефон `+79009267929` во всей микроразметке.

Коллекционные и навигационные страницы могут не иметь JSON-LD. Проверка валидирует все существующие блоки и гарантирует, что микроразметка не исчезла со всего сайта.

## Что запрещено без подтверждённых данных

Проверка блокирует публикацию при появлении:

- `Review`;
- `AggregateRating`;
- `Offer`;
- свойств `review`, `reviews`, `aggregateRating` и `offers`.

Эти элементы можно добавлять только после получения реальных отзывов, разрешения на публикацию и подтверждённых коммерческих условий.

## Запуск

```bash
python tools/check_structured_data.py
```

Проверка входит в `tools/run_quality_checks.py`, поэтому выполняется в каждом PR и перед публикацией GitHub Pages.

## Правила обновления

- При добавлении статьи указывать непустой `headline`, canonical, `mainEntityOfPage`, `datePublished`, `dateModified`, автора и издателя.
- При добавлении услуги указывать `Service.url`, поставщика и телефон; необязательные `serviceType` и `areaServed` не оставлять пустыми.
- В FAQ не дублировать вопросы и не оставлять пустые ответы.
- Не придумывать рейтинг, цену, гарантию, отзыв или статус исполнителя ради расширенной выдачи.
