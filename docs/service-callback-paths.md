# Callback-пути коммерческих и проблемных страниц

Пять страниц с высокой готовностью к обращению получают дополнительный путь к короткой форме обратного звонка:

- `/uslugi/ciklevka-parketa/`;
- `/uslugi/restavraciya-parketa/`;
- `/sovety/parket-posle-vody/`;
- `/sovety/pochemu-skripit-parket/`;
- `/sovety/shcheli-v-parkete/`.

Основные действия не меняются: посетитель по-прежнему сначала видит звонок Ивану и подробную оценку по фото. Обратный звонок добавлен как третий вариант для человека, которому неудобно звонить или заполнять длинную форму.

## Где размещаются ссылки

На каждой странице две статические ссылки:

1. под действиями первого экрана:
   `Неудобно звонить — заказать обратный звонок →`;
2. после второго высокоинтентного CTA-блока:
   `Оставить номер для обратного звонка →`.

Обе ссылки ведут на `/kontakty/#callback`.

## Почему ссылки добавляются при сборке

Исходные страницы имеют плотную однострочную разметку, а одинаковые CTA используются на нескольких материалах. Глобальная текстовая замена могла бы затронуть лишние страницы.

`tools/service_callback_links.py` работает только с пятью точными путями и двумя точными маркерами внутри каждого файла. `tools/build_pages.py` вызывает его после копирования страниц в `_site`.

Если:

- целевая страница отсутствует;
- один из маркеров изменился;
- до сборки присутствует только одна callback-ссылка;
- после вставки получилось не две ссылки;

публичная сборка завершается ошибкой.

## Атрибуция и тема

При переходе на `/kontakty/#callback`:

- форма автоматически прокручивается в область видимости;
- `callback-open` получает `trigger: hash-entry`;
- first-touch UTM сохраняются;
- landing остаётся исходной страницей;
- на один просмотр страницы создаётся не более одного события открытия;
- форма показывает тему обращения по разрешённой внутренней странице;
- Иван получает конкретный текст задачи для циклёвки, реставрации, воды, скрипа или щелей.

Например, переход из объявления Авито на циклёвку сохраняет:

```text
utm_source=avito
utm_medium=classified
utm_campaign=voronezh_parquet_launch
utm_content=service_listing_cyclevka
landing=/uslugi/ciklevka-parketa/
topic=cyclevka
```

Переход со статьи после протечки сохраняет тему `posle-vody`, но не обещает дистанционную диагностику или возможность восстановления без осмотра.

## Проверка

`tests/e2e/service-callback-links.spec.mjs` проверяет:

1. все пять страниц доступны;
2. на каждой ровно две статические ссылки;
3. подписи ссылок совпадают с утверждёнными;
4. переход с циклёвки открывает callback-форму;
5. создаётся одно событие `hash-entry`;
6. UTM Авито и landing страницы услуги сохраняются в browser event и `dataLayer`.

`tests/e2e/callback-topic-context.spec.mjs` дополнительно проверяет конкретные темы и тексты для воды, скрипа и щелей.

## Ограничения

- ссылки появляются в готовой публичной сборке `_site`, а не в исходном HTML;
- исходный отчёт внутренних ссылок не учитывает сборочные overlay-ссылки и остаётся консервативной оценкой графа;
- форма не обещает конкретное время звонка, цену, результат восстановления или выезд;
- при протечке сначала нужно устранить источник воды и дать полу просохнуть без резкого перегрева;
- фактическая отправка зависит от доступного домена и production Edge Function.
