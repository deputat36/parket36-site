# Запуск индексации и поискового обнаружения Паркет36

## Главный вывод

Поисковики не смогут стабильно находить страницы, пока `https://parket36.ru/` не открывается по HTTPS и не отдаёт актуальную сборку GitHub Pages. Поэтому порядок запуска всегда такой:

1. восстановить DNS и публикацию домена;
2. убедиться, что главная, `robots.txt`, `sitemap.xml` и `indexnow-key.txt` доступны с HTTP 200;
3. отправить sitemap и подтвердить сайт в кабинетах поисковиков;
4. подключить аналитику и измерять звонки и заявки;
5. только после этого оценивать позиции и расширять контент.

Чек-лист переключения домена находится в `docs/github-pages-switch-checklist.md`. Ежедневная диагностика выполняется workflow `Live site health`.

## Что автоматизировано

После успешного workflow `Deploy GitHub Pages` запускается workflow `IndexNow`.

Он:

- проверяет `data/indexnow.json`;
- сверяет публичный ключ `indexnow-key.txt`;
- читает все URL из `sitemap.xml`;
- проверяет, что URL принадлежат `parket36.ru`;
- проверяет доступность ключа на боевом домене;
- отправляет один пакет URL в IndexNow;
- пропускает отправку без ошибки, если домен или ключ ещё не опубликованы.

Статическая проверка входит в общий quality gate:

```bash
python tools/submit_indexnow.py --check
```

Проверка ключа на боевом домене:

```bash
python tools/submit_indexnow.py --verify-live-key
```

Ручная отправка всех URL sitemap:

```bash
python tools/submit_indexnow.py --submit
```

Код принимает ответы HTTP 200 и 202 как подтверждение приёма запроса. Это не является гарантией попадания страницы в индекс или высокой позиции.

## Измеряемые ссылки для первых размещений

Готовая матрица ссылок находится в `docs/campaign-links.md`, а исходные каналы — в `data/campaign-links.json`.

В неё входят отдельные URL для:

- публикаций VK;
- объявления по циклёвке на Авито;
- подтверждённых карточек Яндекс Бизнеса и 2ГИС;
- QR-кода на листовке или визитке.

Все ссылки используют одну кампанию и разные `utm_source`, `utm_medium` и `utm_content`. Форма сайта сохраняет эти значения вместе с заявкой. Генератор проверяет существование посадочной страницы, HTTPS-домен, формат параметров, отсутствие дублей и актуальность документа:

```bash
python tools/build_campaign_links.py --check
```

Для изменения канала нужно редактировать `data/campaign-links.json`, затем пересобрать документ:

```bash
python tools/build_campaign_links.py --write
```

До подключения счётчика UTM позволяют различать источники заявок по форме. Для достоверной атрибуции телефонных звонков по кликам всё равно нужна аналитика или отдельное согласованное решение по коллтрекингу.

## Что требуется сделать вручную один раз

### Яндекс Вебмастер

- добавить `https://parket36.ru/`;
- подтвердить права удобным способом;
- отправить `https://parket36.ru/sitemap.xml`;
- проверить диагностику, главное зеркало, HTTPS и страницы в поиске;
- после запуска проверить коммерческие страницы: главную, услуги, циклёвку, реставрацию, стоимость, контакты и заявку.

### Google Search Console

- добавить ресурс домена или URL-префикса;
- подтвердить права;
- отправить sitemap;
- проверить индексирование главной и основных коммерческих страниц;
- не запрашивать ручную индексацию десятков однотипных страниц подряд.

### Bing Webmaster Tools

- добавить сайт и sitemap;
- проверить отчёт IndexNow после первой успешной отправки;
- контролировать ошибки обхода и дубли canonical.

### Локальная видимость

Для быстрых обращений недостаточно только сайта. После подтверждения фактов о мастере нужно создать и заполнить карточки организации или специалиста в доступных локальных каталогах и картах. Готовые тексты, измеряемые ссылки, порядок заполнения и список запрещённых неподтверждённых полей находятся в `docs/local-profile-launch-kit.md`.

В карточках должны совпадать:

- название `Паркет36`;
- имя Иван;
- телефон `8 (900) 926-79-29`;
- регион работы — Воронеж и Воронежская область;
- ссылка `https://parket36.ru/`;
- реальные фотографии;
- фактический график и статус исполнителя.

Нельзя публиковать выдуманный адрес, график, стаж, цены, гарантию, отзывы или юридический статус.

Проверка комплекта локального запуска:

```bash
python tools/build_local_profile_kit.py --check
```

## Как проверять результат

До подключения Метрики минимальный технический контроль:

1. домен открывается без предупреждений;
2. мобильная кнопка звонка вызывает `tel:+79009267929`;
3. форма сохраняет тестовую заявку после production-деплоя Edge Function;
4. `robots.txt` разрешает обход и указывает sitemap;
5. sitemap содержит только canonical URL;
6. IndexNow workflow видит боевой ключ и получает HTTP 200 или 202;
7. поисковые кабинеты начинают показывать обход и индексирование.

После получения ID Яндекс Метрики нужно заполнить `metrika_id` в `data/site.json`, выполнить синхронизацию и проверить цели звонка и заявки по `docs/analytics-events.md`.

## Ограничения текущего запуска

- IndexNow ускоряет передачу URL, но не заменяет доступный домен, sitemap, внутренние ссылки и качество страниц.
- Google не следует считать автоматически охваченным IndexNow; для него нужен Search Console и обычный обход.
- Реальные звонки зависят прежде всего от доступности домена, доверия, коммерческих фактов, локальных карточек и корректной обработки обращений.
