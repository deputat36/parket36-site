# First-touch атрибуция заявок Паркет36

Сайт сохраняет первый источник перехода в `sessionStorage` и использует его на всех внутренних страницах в пределах текущей вкладки.

## Что сохраняется

- `utm_source` — источник;
- `utm_medium` — тип канала;
- `utm_campaign` — кампания;
- `utm_content` — объявление, пост или макет;
- `utm_term` — поисковая фраза, если она была передана;
- `landing` — первая страница входа;
- `firstSeen` — время первого входа.

Следующий UTM-переход в той же вкладке не заменяет первоначальную атрибуцию. Это first-touch модель, а не last-touch.

## Как первая посадочная попадает в заявку

Edge Function уже сохраняет HTTP `Referer` внутри JSON-поля `attribution.referrer`.

Скрипт `js/first-touch-referrer.js` перед отправкой заявки:

1. берёт `landing` из `window.parket36Attribution`;
2. принимает только путь, начинающийся с `/`;
3. создаёт URL только на текущем origin;
4. удаляет query-параметры и fragment;
5. передаёт очищенный URL через настройку Fetch `referrer`;
6. использует `referrerPolicy: unsafe-url`, чтобы путь не был сокращён до одного origin при запросе к Supabase.

Таким образом в `attribution.referrer` сохраняется первая посадочная страница, но не передаются UTM-параметры, якоря или внешний URL.

## Порядок обёрток Fetch

На страницах с формой публичная сборка подключает скрипты в таком порядке:

1. `contact-validation.js`;
2. `lead-reliability.js`;
3. `first-touch-referrer.js`;
4. `main.js`.

`first-touch-referrer.js` вызывает предыдущую обёртку Fetch, поэтому timeout, retry, honeypot и обработка ошибок продолжают работать без изменений.

## Браузерная проверка

`tests/e2e/attribution.spec.mjs` проверяет полный путь:

- вход на главную по готовой VK UTM-ссылке;
- переход на локальную страницу;
- переход к форме оценки;
- сохранение первоначальных UTM в payload;
- передачу первой посадочной в HTTP `Referer` без query и fragment;
- наличие первой страницы входа в копируемом тексте;
- отсутствие перезаписи первой кампании следующим UTM-переходом;
- сохранение first-touch атрибуции при клике по телефону.

## Ограничения

- атрибуция сохраняется только в пределах одной вкладки и одной browser session;
- очистка хранилища, приватный режим или новая вкладка создают новую first-touch запись;
- отправленная форма позволяет связать заявку с каналом, но фактический телефонный разговор требует подключённой аналитики или коллтрекинга;
- HTTP `Referer` может быть изменён расширением браузера или корпоративной политикой приватности, поэтому UTM-поля остаются основным источником данных о канале.
