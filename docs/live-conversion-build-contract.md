# Live-контракт звонка и собранной главной

Дата обновления: 2026-07-16.

## Назначение

`tools/check_live_conversion.py` проверяет не только наличие номера телефона на `parket36.ru`, но и то, что посетителю отдана готовая GitHub Pages-сборка, а не сырой `index.html` или промежуточный CDN-кэш.

Проверка входит в `.github/workflows/live-site-health.yml` и запускается:

- сразу после успешного `Deploy GitHub Pages`;
- ежедневно;
- вручную через `workflow_dispatch`.

## Телефонный маршрут

На боевой главной одновременно должны присутствовать:

- точный `href="tel:+79009267929"` из `data/site.json`;
- отображаемый номер из общих настроек;
- подпись `Позвонить Ивану`;
- действие `Оценка по фото`.

Видимый номер без рабочего `tel:` считается ошибкой.

## Контракт готовой сборки

Главная должна содержать все маркеры, добавляемые только во время формирования `_site`:

- `<!-- shared-shell:header -->`;
- `<!-- shared-shell:final-cta -->`;
- `<!-- shared-shell:footer -->`;
- `<!-- shared-shell:mobile-cta -->`;
- `data-css-bundle="true"`.

Отсутствие хотя бы одного маркера означает, что домен может отдавать сырой исходник, неполную сборку или устаревший HTML. Такой результат делает live workflow красным даже при сохранённом номере телефона.

## IndexNow-ключ

Проверяется:

- HTTP 200 для публичного файла ключа;
- точное совпадение содержимого с `data/indexnow.json`;
- отсутствие вывода самого ключа в диагностический отчёт.

## Защита от CDN-кэша

Каждая попытка использует уникальные query-параметры:

- `verify_conversion` для главной;
- `verify_indexnow_key` для файла ключа;
- `attempt` с номером попытки.

Запросы дополнительно отправляют:

```text
Cache-Control: no-cache, no-store, max-age=0
Pragma: no-cache
```

Post-deploy режим выполняет до шести попыток с интервалом 10 секунд. Ежедневный и ручной запуск выполняют одну попытку.

## Диагностика

Результаты дописываются в общий `live-health-report.md`. Для каждого HTTP-запроса указывается номер `cache_bust_attempt`, но случайный nonce и содержимое IndexNow-ключа в отчёт не выводятся.

Если проверка не прошла, artifact всё равно сохраняется. После повторного сбоя существующий механизм monitoring issue создаёт или обновляет задачу `[monitoring] parket36.ru live health failure`.

## Офлайн-проверки

```bash
python tools/check_live_conversion.py --self-test
python tools/check_live_conversion_workflow.py
```

Self-test проверяет рабочие и неполные телефонные маркеры, полный и повреждённый shared-shell контракт, точное содержимое ключа, уникальные URL главной и ключа, а также ограничения runtime-параметров.

## Границы

Проверка ничего не публикует и не изменяет на боевом сайте. Она не отправляет заявки, не вызывает Supabase Edge Function и не подтверждает поисковую индексацию. Её задача — доказать, что live-домен отдаёт готовую конверсионную сборку и публичный ключ владения IndexNow.
