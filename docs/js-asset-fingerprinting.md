# Fingerprinting JavaScript

Дата обновления: 2026-07-15.

## Цель

Публичная сборка Паркет36 не должна зависеть от устаревшего браузерного кэша после изменения логики форм, аналитики, меню или интерфейса.

Исходные HTML и JavaScript не изменяются. Fingerprinting выполняется только внутри `_site` после подготовки всех служебных скриптов и перед проверкой публичных ссылок.

## Как работает сборка

`python tools/build_pages.py`:

1. копирует каталог `js/` в `_site/js/`;
2. добавляет на страницы форм обязательные скрипты надёжности и обратной связи;
3. вычисляет первые 12 символов SHA-256 содержимого каждого JavaScript-файла;
4. переименовывает файл, например:

```text
/js/main.js → /js/main.a1b2c3d4e5f6.js
```

5. переписывает ссылки только в собранных HTML-файлах;
6. сохраняет исходный порядок `defer`-скриптов;
7. проверяет, что каждый опубликованный JS-файл имеет корректный fingerprint и существует по указанному URL.

При изменении содержимого меняется имя файла. Браузер получает новый URL и не использует старую версию из кэша.

## Что не меняется

- исходные файлы в `js/` сохраняют обычные понятные имена;
- исходные HTML продолжают ссылаться на `/js/main.js`, `/js/callback-form.js` и другие читаемые пути;
- порядок выполнения скриптов не меняется;
- атрибуты `defer` сохраняются;
- production endpoint, Supabase secrets и данные заявок не используются;
- JavaScript не объединяется в один файл, поэтому независимые модули остаются удобными для поддержки и диагностики.

## Fail-closed проверки

Сборка завершается ошибкой, если:

- HTML ссылается на отсутствующий JavaScript-файл;
- ссылка содержит ручной query-параметр или fragment;
- исходный файл уже имеет вид fingerprinted-файла;
- рассчитанный SHA-256 не совпадает с хэшем в имени;
- после сборки остаётся ссылка на незаверсионированный `/js/*.js`;
- fingerprinted-файл отсутствует в `_site`.

## Автоматическая проверка

Self-test:

```bash
python tools/js_assets.py --self-test
```

Он проверяет:

- два скрипта, включая файл во вложенном каталоге;
- сохранение порядка `defer`-скриптов;
- удаление незаверсионированных копий из `_site`;
- соответствие имени фактическому содержимому;
- смену валидности fingerprint после изменения содержимого;
- отказ при ссылке на отсутствующий файл.

Статический guardrail:

```bash
python tools/check_js_assets.py
```

Проверка входит в общий quality gate. После изменения также выполняются Browser smoke и Lighthouse CI, поэтому неправильная ссылка, нарушение порядка загрузки или ошибка исполнения остановят PR до слияния и публикации Pages.
