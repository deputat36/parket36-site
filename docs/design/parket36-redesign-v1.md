# Паркет36 — новая дизайн-концепция v1

## Рабочее пространство Figma

Файл: https://www.figma.com/design/2ovBluMs8xOKkkUIPevLaH

Структура файла:

1. `Cover + Brand — Паркет36`
2. `Foundations + Components — Дизайн-система`
3. `Screens — Desktop и Mobile сайт`

Из-за ограничений Starter-плана Figma рабочая структура собрана на трёх страницах. Компоненты, основы и экраны должны размещаться внутри этих страниц отдельными секциями.

## Цель редизайна

Создать более стильный, привлекательный, понятный и удобный сайт, который воспринимается не как типовой сайт строительной услуги, а как аккуратный экспертный сервис по восстановлению деревянных полов.

Основное позиционирование:

> Мастер помогает сохранить и восстановить ценный деревянный пол, когда это технически возможно, а не предлагает замену по умолчанию.

## Что сохраняется из текущего сайта

Редизайн использует функциональную и содержательную основу текущего проекта:

- звонок мастеру;
- предварительная оценка по фотографиям;
- выбор услуги по проблеме клиента;
- честное разделение сохранения заявки и подтверждённого уведомления;
- услуги по паркету и деревянным полам;
- решения для собственников, риелторов и объектов после арендаторов;
- портфолио и реальные примеры задач;
- советы и SEO-статьи;
- страницы стоимости, контактов, мастера и формы заявки;
- адаптивная мобильная панель быстрых действий.

Текущая визуальная стилистика не является основой нового дизайна.

## Новое визуальное направление

### Характер бренда

- профессиональный;
- спокойный;
- точный;
- ремесленный, но не кустарный;
- современный;
- локальный;
- честный;
- визуально близкий к интерьерным и архитектурным брендам.

### Визуальные принципы

- много свободного пространства;
- крупная типографика;
- спокойные нейтральные поверхности;
- глубокий хвойный зелёный как главный цвет доверия;
- латунный акцент для тёплых деталей и премиальности;
- минимум декоративного шума;
- реальные фотографии дерева и работ вместо абстрактных строительных изображений;
- понятный сценарий «проблема → диагностика → решение → действие»;
- CTA не конкурируют друг с другом;
- основной путь — оценка по фото, альтернативный — звонок.

## Логотип

Логотип создаётся с нуля и не повторяет текущий знак `36`.

### Основная идея

Знак объединяет:

- торцевой срез дерева или годовое кольцо;
- геометрию паркетной планки;
- число `36` как региональный идентификатор;
- ощущение точности и ручной работы.

### Требования

- узнаваемость при размере 24–32 px;
- отдельный знак для favicon и аватара;
- горизонтальная версия со словом `Паркет36`;
- монохромная версия;
- версия на светлом и тёмном фоне;
- отсутствие сложных мелких деталей;
- отсутствие прямого копирования распространённых иконок дома, молотка, пилы или дерева целиком.

### План вариантов

1. Премиальный минимализм — геометрический знак из паркетных планок и числа 36.
2. Ремесленная точность — кольцо дерева с аккуратным цифровым модулем.
3. Современный локальный бренд — компактная монограмма `П36`.

## Типографика

- заголовки и крупные акценты — `Manrope`;
- основной текст, формы и интерфейс — `Onest`.

Оба семейства поддерживают кириллицу и доступны в текущем Figma-пространстве.

### Базовая шкала

- Display — 64 px, Manrope ExtraBold;
- H1 — 52 px, Manrope ExtraBold;
- H2 — 40 px, Manrope Bold;
- H3 — 28 px, Manrope Bold;
- Lead — 20 px, Onest Regular;

- Body Strong — 16 px, Onest SemiBold;
- Small — 14 px, Onest Regular;

Мобильные размеры должны снижаться плавно, без резкого уменьшения и без строк заголовка шире 12–14 слов.

## Главная страница Desktop

Предлагаемая последовательность:

1. Шапка с логотипом, короткой навигацией, телефоном и кнопкой оценки.
2. Hero: сильный заголовок, честное обещание, два действия и крупный визуальный блок «до / после».
3. Быстрый выбор проблемы: лак, скрип, щели, локальный ремонт, оценка по фото.
4. Блок «Можно ли восстановить ваш пол» с диагностическим подходом.
5. Основные услуги, сгруппированные по задаче клиента.
6. Примеры работ с коротким описанием исходного состояния и результата.
7. Этапы работы: фото → уточнение → осмотр при необходимости → работы → рекомендации.
8. Блок мастера Ивана с реальной фотографией и принципами работы.
9. Стоимость без ложной точности: факторы цены и пример расчёта после диагностики.
10. Форма оценки по фото.
11. FAQ.
12. Финальный CTA и контакты.
13. Содержательный footer.

## Главная страница Mobile

Приоритеты:

- телефон и оценка доступны без поиска по меню;
- первый экран укладывается в понятный сценарий за 5–7 секунд;
- заголовок не занимает весь экран;
- действия не перекрывают контент;
- карточки идут одной колонкой;
- формы используют крупные поля и правильную клавиатуру;
- статус заявки и fallback остаются видимыми над фиксированной панелью;
- фотография или пример работы не вытесняют CTA ниже первого экрана полностью.

## Страница мастера

Основные блоки:

- профессиональный портрет;
- короткая история и специализация;
- принципы честной диагностики;
- что мастер делает лично;
- география работ;
- примеры задач;
- отзывы после получения реальных материалов;
- контакты.

Не указывать неподтверждённый стаж, гарантию или количество выполненных объектов.

## Форма оценки по фото

Форма должна ощущаться не как длинная анкета, а как помощник диагностики.

Рекомендуемая структура:

1. Что происходит с полом.
2. Где находится объект.
3. Примерная площадь.
4. Какие фотографии уже готовы.
5. Короткое описание желаемого результата.
6. Контакт и удобное время.
7. Ясное описание следующего шага.

Сценарии статуса:

- заявка сохранена и уведомление подтверждено;
- заявка сохранена, уведомление не подтверждено;
- заявка не сохранена, текст скопирован;
- clipboard недоступен, показан ручной текст;
- повторная заявка;
- ошибка поля.

## Изображения

До получения реальных материалов использовать только нейтральные заглушки и схемы.

Не публиковать изображения, создающие впечатление реальной работы мастера, если источник не подтверждён.

Для полноценного дизайна нужны:

- 10–20 реальных фото работ;
- 3–5 пар «до / после»;
- портрет мастера;
- детали паркета крупным планом;
- фотографии инструмента и рабочего процесса;
- 3–5 подтверждённых отзывов.

## Доступность

- контраст текста не ниже WCAG AA;
- зона нажатия не меньше 44 × 44 px;
- видимый focus-state;
- цвет не является единственным индикатором состояния;
- формы имеют label, help и error;
- динамические статусы доступны для экранных дикторов;
- мобильная фиксированная панель не перекрывает fallback-действия;
- уменьшение анимации учитывает `prefers-reduced-motion`;
- интерактивные элементы имеют понятные названия.

## Связь с кодом

Текущий проект использует чистые HTML, CSS и JavaScript без UI-фреймворка и без Code Connect. Поэтому:

- компоненты Figma создаются как проектная дизайн-система;
- токены получают CSS-совместимые имена;
- после утверждения макетов значения переносятся в отдельный CSS token layer;
- существующая логика форм, аналитики и fallback не переписывается во время визуального редизайна;
- внедрение выполняется по разделам, с Browser smoke, Lighthouse и Site quality после каждого PR.
