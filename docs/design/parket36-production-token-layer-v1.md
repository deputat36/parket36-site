# Паркет36 — production-слой дизайн-токенов v1

## Назначение

Первый этап внедрения новой дизайн-системы в действующий сайт является визуально нейтральный: публичная CSS-сборка получает набор токенов, но существующие селекторы пока продолжают использовать прежние значения.

Это позволяет внедрять новый дизайн небольшими проверяемыми PR без одновременной замены всей вёрстки, форм и аналитики.

## Источник истины

Исходные значения хранятся в:

- `design/parket36-tokens.json`.

Скрипт `tools/build_design_token_css.py` создаёт две байт-в-байт одинаковые копии:

- `design/generated/parket36-tokens.css` — для внутренних прототипов и Figma-синхронизации;
- `css/design-tokens.css` — для production CSS-бандла.

Редактировать сгенерированные CSS-файлы вручную запрещено.

## Production-подключение

`css/design-tokens.css` подключается первым модулем в `CSS_MODULES` файла `tools/css_bundle.py`.

После него собираются действующие стили сайта. Поэтому существующие правила сохраняют приоритет и внешний вид сайта на этом этапе не меняется.

Слой содержит 80 CSS-переменных с префиксом `--p36-`:

- примитивные и семантические цвета;
- интервалы;
- радиусы;
- базовые размеры;
- семейства, размеры и начертания шрифтов;
- тени.

## Ограничение этапа v1

Пока ни один действующий CSS-модуль не должен использовать `var(--p36-...)`. Это временный guardrail, подтверждающий отсутствие визуального влияния инфраструктурного PR.

Следующий визуальный PR должен:

1. выбрать один ограниченный компонент;
2. заменить его значения на токены;
3. обновить guardrail и документацию;
4. пройти Browser smoke, axe и Lighthouse;
5. не менять логику заявок, телефона, аналитики и Supabase.

Рекомендуемый первый компонент — кнопки и focus-state, потому что их контракт уже зафиксирован в `design/components/parket36-components.json`.

## Проверка

```bash
python tools/build_design_token_css.py --check
python tools/check_production_design_token_layer.py
python tools/build_pages.py
```

Проверка должна подтверждать:

- обе CSS-копии совпадают;
- объявлено ровно 80 переменных;
- `design-tokens.css` расположен первым модулем;
- токены пока не используются другими production CSS-файлами;
- публичная сборка содержит один cache-busted CSS-бандл.
