## Разрешённые конфликты

### Структура страниц

Рекомендуемая библиотечная структура предполагает отдельные страницы для компонентов. На Starter-плане файл уже использует три доступные страницы. Решение: не создавать новые страницы; Foundations и компоненты размещать отдельными секциями и фреймами на странице `2:4`.

### Источник истины

До повторной Figma-верификации GitHub-контракты являются источником спецификации, но не окончательной визуальной истиной. После создания и screenshot QA подтверждённые node IDs возвращаются в sync manifest.

### Существующие переменные

Переменные не пересоздаются вслепую. Сначала выполняется read-only аудит. Совпадающие значения обновляются и дополняются scopes/code syntax; отсутствующие создаются; конфликтующие фиксируются отдельно.

### Логотип

Концепция A используется в каталогах только как рабочий маркер. Она не считается финальным production-логотипом до визуального сравнения вариантов и проверки на малых размерах.
