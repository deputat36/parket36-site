# Паркет36 — миграция Problem Card на токены v1

## Область

На компонент Problem Card переведены только четыре ссылки внутри `.quick-choice__grid` на главной странице.

Компонент помогает посетителю выбрать ближайшую проблему с полом и перейти к подходящей услуге, совету или форме оценки.

Другие карточки сайта не изменяются.

## Соответствие контракту

Контракт хранится в `design/components/parket36-components.json`.

Анатомия production-компонента:

- Number — CSS-счётчик `01–04` через `a::before`;
- Title — существующий `<strong>`;
- Description — существующий `<span>`;
- Arrow — существующий `strong::after`;
- Container — вся ссылка `.quick-choice__grid a`.

Вся карточка остаётся кликабельной. Адреса ссылок и тексты не изменяются.

## Размеры

- минимальная высота — 168 px;
- внутренние поля — 24 px;
- расстояние между элементами — 12 px;
- радиус — `radius.lg`, 20 px;
- размер номера — 14 px;
- размер заголовка — 20 px;
- размер описания — 16 px.

## Состояния

### Default

Белая поверхность, нейтральная граница и тень `shadow.card`.

### Hover

Карточка поднимается на 4 px, получает forest-subtle фон, усиленную границу и `shadow.floating`.

### Focus

Используется видимый brass focus-ring толщиной 3 px с отступом 3 px.

## Доступность

- вся карточка остаётся ссылкой;
- focus-visible не зависит только от изменения цвета;
- заголовок и описание остаются текстом документа;
- CSS-счётчик является декоративным и не заменяет название ссылки;
- мобильная раскладка остаётся одноколоночной через существующее правило до 640 px;
- reduced motion продолжает отключать смещения через существующий accessibility-слой.

## Не изменяется

- HTML главной страницы;
- тексты и href;
- кнопки и Badge;
- JavaScript;
- формы и payload;
- телефон и аналитика;
- Supabase;
- карточки услуг, сценариев, цен и доказательств.
