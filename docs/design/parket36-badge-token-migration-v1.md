# Паркет36 — миграция Badge на токены v1

## Цель

Перевести короткие признаки доверия `.trust span` на утверждённый компонент `Badge`, не меняя тексты, HTML и пользовательские сценарии.

Источники:

- `design/components/parket36-components.json`;
- `design/parket36-tokens.json`.

## Область применения

Миграция касается только элементов внутри `.trust`.

Они используются для коротких подтверждающих признаков, например:

- специализация на паркете;
- возможность предварительной оценки по фото;
- отсутствие обещаний до диагностики.

Интерактивные `.pill`, статусы формы и другие округлые элементы не являются Badge и в этот PR не входят.

## Вариант

Production использует тон `forest`:

- фон — `color.semantic.surface.subtle`;
- текст — `color.semantic.text.accent`;
- граница — `color.semantic.border.strong`.

Тоны `brass` и `neutral` остаются в компонентном контракте и не внедряются без подтверждённого сценария.

## Размеры

- минимальная высота — 34 px;
- вертикальные поля — 7 px;
- горизонтальные поля — 12 px;
- радиус — `radius.full`;
- размер текста — `font.size.small`, 14 px;
- насыщенность — `font.weight.semibold`, 600.

## Адаптивность

На desktop Badge остаются в одной переносимой flex-строке.

На mobile до 640 px сохраняется существующее поведение:

- ширина 100%;
- выравнивание по центру;
- текст по центру.

## Что не меняется

- тексты признаков доверия;
- порядок элементов;
- HTML страниц;
- кнопки и ссылки;
- формы;
- JavaScript;
- аналитика;
- Supabase.

## CSS-порядок

Токенизированный блок расположен в конце `css/cta-polish.css`, после базового `.trust span` из `style.css` и mobile-уточнений из `interface-polish.css`.

Новый блок уточняет визуальные значения, но не переопределяет mobile-ширину и центрирование.
